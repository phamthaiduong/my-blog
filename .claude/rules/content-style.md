# Content Style Rules

Áp dụng khi tạo hoặc edit file .mdx trong data/blog/

## Song ngữ
- Bài song ngữ: phần VI trước, kẻ ngang `---`, phần EN sau
- Heading level giữ nhất quán giữa 2 phần
- Không dịch máy móc — giữ tone tự nhiên của từng ngôn ngữ

## Markdown
- Heading tối đa H3 trong bài viết (H1 dành cho title)
- Code block phải có language tag: ```javascript, ```python
- Link external dùng absolute URL

## Frontmatter
- date luôn là ngày thực tế publish, không để future date
- tags tối đa 5 tags mỗi bài
- summary 1-2 câu, không quá 160 ký tự
